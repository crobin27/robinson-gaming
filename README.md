# Robinson Gaming Portfolio Website\n\n## Overview\nThis project is a personal portfolio website for Robinson Gaming, developed to showcase various projects, including mobile games, web applications, and IoT solutions. The website is built using **Astro.js** for a fast, static site, with components styled using **CSS** and **TailwindCSS**. It is hosted on **Cloudflare Pages** with CI/CD integration and includes media storage via **Cloudflare R2**. The project demonstrates modern web development practices, CI/CD workflows, and efficient asset management.\n\n## Features\n- **Astro.js Framework**: Static site generation using Astro.js for fast, performance-optimized delivery.\n- **Cloudflare R2 Integration**: Utilizes Cloudflare R2 as a media storage solution for downloadable assets, allowing for efficient access to larger files.\n- **Responsive Design**: Fully responsive layout built with custom CSS and TailwindCSS, ensuring a great user experience across devices.\n- **CI/CD Pipeline**: Automated deployment through Cloudflare Pages, triggered on Git push for streamlined updates.\n- **CORS Configuration**: Configured CORS policy on Cloudflare R2 to allow secure access to assets only from the portfolio domain.\n- **SEO Optimized**: Meta tags, headers, and structured data optimized for search engines.\n\n## Project Structure\n`\n/RobinsonGamingPortfolio\n├── public/\n│   ├── images/                     # Static image                    # Project documentation (this file)\n└── astro.config.mjs                # Astro configuration file\n`\n\n## Dependencies\nThis project uses a combination of front-end frameworks, storage solutions, and deployment services:\n- **Astro.js**: For static site generation and component-based architecture.\n- **TailwindCSS**: For utility-based styling.\n- **Cloudflare Pages**: For hosting and CI/CD.\n- **Cloudflare R2**: For scalable, affordable file storage, especially for downloadable assets.\n- **Git**: Version control with GitHub as the remote repository.\n\n## Getting Started\nTo set up this project locally, follow these steps:\n\n1. **Clone the Repository**:\n `bash\n   git clone https://github.com/username/RobinsonGamingPortfolio.git\n   cd RobinsonGamingPortfolio\n   `\n\n2. **Install Dependencies**:\n `bash\n   npm install\n   `\n\n3. **Run Locally**:\n `bash\n   npm run dev\n   `\n Visit `http://localhost:3000` to view the site locally.\n\n4. **Environment Configuration**:\n - **Cloudflare R2**: Set up your Cloudflare R2 bucket and configure CORS policies to allow access only from your domain.\n - **Cloudflare Pages**: Connect this repository to Cloudflare Pages for automatic deployment on every push.\n\n## Key Features Implemented\n- **Responsive Layouts**: CSS media queries ensure layouts adapt to different screen sizes.\n- **CORS Policies for Secure Downloads**: Cloudflare R2 configured to allow downloads only from the portfolio domain.\n- **SEO & Accessibility**: Meta tags, structured data, and alt text on images improve SEO and accessibility.\n- **Modular Components**: Reusable components in Astro for headers, footers, and project displays.\n- **Custom Privacy Policy**: Dedicated page for the privacy policy, styled according to legal requirements.\n\n## CI/CD Process\n- **Continuous Deployment**: GitHub is configured to push changes automatically to Cloudflare Pages, triggering a new build and deployment.\n- **Automated Build**: Cloudflare Pages runs a build using Astro’s static site generator upon every push.\n- **Branch-Based Deployments**: Deployments can be previewed on separate branches to review changes before going live.\n\n## Future Improvements\n- **Dynamic Content Loading**: Implement client-side loading for certain assets to improve initial load time.\n- **Enhanced
